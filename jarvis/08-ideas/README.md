# Ideas

This is the brainstorming space for JARVIS. Ideas live here in three stages:

1. **Backlog** (`backlog.md`) — Raw ideas, half-baked thoughts, "what if we could..." notes. No filter. Dump everything here.
2. **Exploring** (`exploring/`) — Ideas we're actively researching. Each gets its own file with feasibility notes, technical approach, and open questions.
3. **Accepted** — Moves into the relevant module folder (01-the-brain, 04-the-hands, etc.) and onto the ROADMAP.

## How to use this

- ET drops ideas into backlog.md anytime (voice note, text, whatever)
- We review the backlog together periodically
- Pick one idea, research feasibility, move it to exploring/
- If it's viable, plan it and move to ROADMAP

## The core question

What can a local LLM on a Jetson do that Alexa/Google Home/Siri literally cannot?

The answer is: **reason, remember, and integrate freely** — no cloud lock-in, no corporate guardrails, no "sorry, I can't do that" for things that are technically trivial.
